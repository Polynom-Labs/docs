---
title: "Withdrawals"
description: "Move private balance to a public chain address or payout destination."
---

A `Withdrawal` spends private balance and sends funds to a public destination.

<Warning>
These endpoint examples describe the intended hosted API model. If you are integrating through the SDK today, implement equivalent resources in your backend and confirm hosted API availability with Arcane before coding against these routes.
</Warning>

## Object

```json
{
  "id": "wd_...",
  "status": "preparing",
  "private_account_id": "pa_...",
  "amount": "50.00",
  "asset": "USDC",
  "chain": "solana",
  "destination_address": "...",
  "external_reference": "withdrawal_123",
  "created_at": "2026-06-25T10:00:00Z"
}
```

## Create a withdrawal

```http
POST /withdrawals
```

```json
{
  "private_account_id": "pa_...",
  "amount": "50.00",
  "asset": "USDC",
  "chain": "solana",
  "destination_address": "...",
  "external_reference": "withdrawal_123"
}
```

## Retrieve a withdrawal

```http
GET /withdrawals/{id}
```

## Statuses

| Status | Meaning |
| --- | --- |
| `requires_balance` | Private balance is insufficient |
| `requires_review` | Product or compliance review is required |
| `preparing` | Arcane is preparing proof and transaction data |
| `submitted` | Transaction was submitted |
| `confirmed` | Public chain confirmed the transaction |
| `indexed` | Arcane indexed the private spend |
| `paid_out` | Your product marked the payout complete |
| `failed` | Retry or review is required |
