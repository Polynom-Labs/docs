---
title: "Balances"
description: "Read available private balances for accounts and product scopes."
---

Balances represent indexed private state that is available for product use. Do not treat a chain-confirmed transaction as spendable until Arcane has indexed the resulting private state.

<Warning>
These endpoint examples describe the intended hosted API model. If you are integrating through the SDK today, implement equivalent resources in your backend and confirm hosted API availability with Arcane before coding against these routes.
</Warning>

## Retrieve account balances

```http
GET /private-accounts/{id}/balances
```

## Response

```json
{
  "private_account_id": "pa_...",
  "balances": [
    {
      "asset": "USDC",
      "chain": "solana",
      "available": "100.00",
      "pending": "0.00"
    }
  ],
  "indexed_at": "2026-06-25T10:00:00Z"
}
```

## Balance states

| State | Meaning |
| --- | --- |
| `available` | Indexed private balance that can be spent |
| `pending` | Funding, transfer, or withdrawal activity that is not fully indexed |
| `held` | Balance held by product or compliance policy |

## Product guidance

- Show available balance for user actions.
- Show processing status for pending activity.
- Re-scan before transfers and withdrawals.
- Keep product ledger balances separate from Arcane indexed balances.
