---
title: "Private Transfers"
description: "Move private balance between accounts or recipients."
---

A `PrivateTransfer` moves value inside the Arcane privacy layer.

<Warning>
These endpoint examples describe the intended hosted API model. If you are integrating through the SDK today, implement equivalent resources in your backend and confirm hosted API availability with Arcane before coding against these routes.
</Warning>

## Object

```json
{
  "id": "pt_...",
  "status": "preparing",
  "from_private_account_id": "pa_...",
  "to_private_account_id": "pa_...",
  "amount": "25.00",
  "asset": "USDC",
  "external_reference": "order_123",
  "created_at": "2026-06-25T10:00:00Z"
}
```

## Create a private transfer

```http
POST /private-transfers
```

```json
{
  "from_private_account_id": "pa_sender_...",
  "to_private_account_id": "pa_recipient_...",
  "amount": "25.00",
  "asset": "USDC",
  "external_reference": "order_123"
}
```

## Retrieve a private transfer

```http
GET /private-transfers/{id}
```

Retrieve transfers until the status reaches `available` or `failed`.

## Statuses

| Status | Meaning |
| --- | --- |
| `requires_balance` | Sender has insufficient available balance |
| `preparing` | Arcane is preparing inputs, outputs, and proof data |
| `submitted` | Transaction was submitted |
| `confirmed` | Chain confirmed the transaction |
| `indexed` | Arcane indexed new private state |
| `available` | Recipient balance is usable |
| `failed` | Retry or review is required |
