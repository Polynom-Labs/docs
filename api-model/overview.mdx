The pages in this section describe the intended partner-facing resource model for Arcane integrations.

<Warning>
These pages are not a generated OpenAPI reference yet. If you are integrating through the SDK today, implement equivalent resources in your backend and confirm hosted API availability with Arcane before coding against these HTTP routes.
</Warning>

## Why this model exists

Arcane integrations should be designed around product-level resources instead of protocol internals.

| Resource | Purpose |
| --- | --- |
| `PrivateAccount` | Private balance scope for a user, cardholder, employee, treasury, or application |
| `FundingIntent` | Public-to-private funding action |
| `PrivateTransfer` | Private movement between private accounts or recipients |
| `Withdrawal` | Private-to-public movement |
| `Balance` | Indexed private balance view |
| `AuditRecord` | Permissioned record for reconciliation and disclosure |
| Webhook event | Asynchronous status update |
