---
title: "Audit Records"
description: "Reference private activity through controlled, permissioned records."
---

An `AuditRecord` links Arcane private activity to a permissioned review path. It does not mean every user or operator can see all private transaction details.

<Warning>
These endpoint examples describe the intended hosted API model. If you are integrating through the SDK today, implement equivalent resources in your backend and confirm hosted API availability with Arcane before coding against these routes.
</Warning>

## Object

```json
{
  "id": "ar_...",
  "resource_type": "funding_intent",
  "resource_id": "fi_...",
  "external_reference": "card_load_123",
  "chain": "solana",
  "asset": "USDC",
  "created_at": "2026-06-25T10:00:00Z"
}
```

## Retrieve an audit record reference

```http
GET /audit-records/{id}
```

The response should respect the caller's organization, application, case, and permission scope.

## Disclosure use

Use audit records for:

- Support investigations.
- Disputes.
- Payroll review.
- Card funding reconciliation.
- Compliance reports.
- Regulator or auditor requests.

## Access model

Audit details should be accessed through approved disclosure workflows. See:

- [Controlled Disclosure](/integration-guides/controlled-disclosure)
- [Disclosure Cases and Reports](/auditing-portal/disclosure-cases-and-reports)
