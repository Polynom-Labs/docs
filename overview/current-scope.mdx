These docs describe the current Arcane compliance and selective-disclosure platform for the Stellar/Soroban privacy-pool implementation.

## In scope

| Area | Covered implementation |
| --- | --- |
| On-chain privacy pool | Soroban `PrivacyPoolsContract`, `transact` entrypoint, proof verification, storage, token movement, and audit event emission |
| Cryptography | Circom transaction circuit model, commitments, nullifiers, roots, stealth addresses, and SDK cryptographic helpers |
| Client integration | `@auditable/privacy-pool-zk-sdk`, wallet signing boundary, proof generation, public signals, and Soroban call serialization |
| Audit event boundary | `AuditEncodedDigest` event and encrypted audit payload handoff |
| Backend indexing | Stellar scanner, registered contracts, raw `audit` rows, scan checkpoints, and interpretation batch processing |
| Backend data model | PostgreSQL entities for contracts, applications, audit rows, interpretation rows, cases, reports, permissions, and logs |
| Auditing Portal | React Audit UI, backend API, auth, permissions, application scope, case workflows, reports, and activity logs |
| Use cases | Auditor, application administrator, and organization administrator workflows |

## Chain scope

The privacy-pool pages focus on Stellar/Soroban.

The backend also contains chain-adapter concepts, including Solana confidential-token scanner and interpretation modules. Those modules share backend workflow surfaces after normalization, but they are not the primary privacy-pool implementation described in this documentation set.

## Out of scope

| Area | Status |
| --- | --- |
| Full Solana confidential-token integration guide | Not covered in this documentation pass |
| Production infrastructure runbooks | Not covered beyond architecture and integration behavior |
| End-user wallet UX design | Application-owned integration surface |
| Custody, key management, or wallet hosting | Outside Arcane's current product boundary |
| Identity-provider administration guide | The backend integrates identity providers; provider-specific setup is not the core docs focus |
| Legal policy, compliance interpretation, or regulatory advice | Outside technical documentation scope |
