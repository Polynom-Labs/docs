---
title: "Controlled Disclosure"
description: "Expose private transaction data only through scoped, authorized review workflows."
---

Arcane is designed for privacy-preserving applications that still need a controlled path for compliance review, dispute resolution, and reporting.

Controlled disclosure means private transaction data is not broadly visible by default. Authorized users request, approve, review, and report only the scoped data they are allowed to see.

## Disclosure flow

```mermaid
sequenceDiagram
  participant Auditor
  participant Admin as Application admin
  participant Portal as Arcane Auditing Portal
  participant Records as Audit records

  Auditor->>Portal: Create disclosure request
  Portal->>Admin: Notify for approval
  Admin->>Portal: Approve scope and access window
  Auditor->>Portal: Review approved case
  Portal->>Records: Read scoped audit records
  Portal-->>Auditor: Show permitted fields
```

## Scope dimensions

Disclosure should be limited by scope.

| Dimension | Examples |
| --- | --- |
| Organization | Which partner or customer organization owns the data |
| Application | Which app, card program, payroll product, or treasury product |
| Time window | Start and end time for permitted review |
| Asset | USDC, SOL, native token, or supported private asset |
| Operation references | Product object ids, transaction signatures, disclosure cases, and reports |
| Role | Auditor, application administrator, organization administrator |
| Case | Specific investigation, dispute, report, or legal request |

## What disclosure is for

Use controlled disclosure for:

- Support investigations.
- Card disputes and chargeback research.
- Payroll disputes.
- Fraud review.
- Regulatory requests.
- Internal compliance reports.
- External audits.

Do not use controlled disclosure as a general analytics export. Build product analytics from partner-side product events and aggregate Arcane status data.

## Integration requirements

Your backend should:

- Store product references and transaction signatures for each relevant product action.
- Keep product references stable and searchable.
- Preserve status history.
- Route sensitive review through the Auditing Portal or an equivalent scoped workflow.
- Log access, report generation, and downloads.
