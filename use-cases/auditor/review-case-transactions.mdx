---
title: "Review Case Transactions"
description: "How assigned auditors review interpreted transaction data in an approved investigation case."
---

After an application administrator approves a disclosure case, assigned auditors review interpreted transactions within the approved scope and period. Fields outside the approved scope are not returned or are displayed as restricted.

## Prerequisites

- Case assignment for the investigation case.
- `reports:view_transactions`.
- Case status is approved.
- Case access window has not expired.

## Workflow

<Steps>
  <Step title="Open Case Review">
    Sign in and select the application workspace. In the sidebar, click **Case Review**.
  </Step>
  <Step title="Select an approved case">
    Use search and status filters to find an approved case assigned to you. Open the case row.
  </Step>
  <Step title="Review transactions">
    Open the **Review** tab and inspect the transaction table for the approved period.
  </Step>
  <Step title="Inspect disclosure effects">
    Check transaction id, sender, and recipient/withdrawal fields. Values outside the approved scope remain hidden.
  </Step>
</Steps>

## Query scope

Case transaction review reads `audit_interpretation` after these filters:

| Filter | Source |
| --- | --- |
| Organization | Authenticated session |
| Application | `:foreignId` resolved by `ApplicationScopeGuard` |
| Case | `cases.id` |
| Period | `case_requests.period_from`, `case_requests.period_to` |
| Contract address | Request or case contract filters |
| Assignment | `case_auditor_assignments` |
| Access window | `cases.created_at + access_days` |
| Field scope | Disclosure flags |

## Related actions

From the case workspace:

- Open **Reports** to generate transaction summary exports.
- Open **Case Auditors** to view assignments.
- Open **Activity** to view case-scoped activity log entries.
