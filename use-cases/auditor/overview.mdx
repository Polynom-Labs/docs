Auditors work inside an application workspace. Their access to interpreted transaction data is case-scoped and time-bound.

## Main use cases

| Use case | Required permission | Page |
| --- | --- | --- |
| Create a disclosure request | `cases:create` | [Create a disclosure request](/use-cases/auditor/create-disclosure-request) |
| Withdraw own pending request | `cases:withdraw_pending_request` | [Create a disclosure request](/use-cases/auditor/create-disclosure-request) |
| Review assigned case transactions | `reports:view_transactions` plus active assignment | [Review case transactions](/use-cases/auditor/review-case-transactions) |
| Generate case transaction report | `reports:create` | [Generate reports and activity logs](/use-cases/auditor/generate-reports-and-activity-logs) |
| Download reports | `reports:download` | [Generate reports and activity logs](/use-cases/auditor/generate-reports-and-activity-logs) |
| View case activity | `reports:view_transactions` | [Generate reports and activity logs](/use-cases/auditor/generate-reports-and-activity-logs) |

## Access model

Auditor access is controlled by:

- Application permission bucket from `team_member_application_permissions.auditor_permissions`.
- Application route scope from `applications.foreign_id`.
- Case assignment from `case_auditor_assignments`.
- Case period and access window from `case_requests` / `cases`.
- Disclosure flags such as `full_tx_ids`, `sender_information`, and `withdrawal_details`.

## Primary route family

```text
/workspace/application/:foreignId/cases
/workspace/application/:foreignId/cases/new
/workspace/application/:foreignId/cases/:caseId
```
