---
title: "Approve a Disclosure Request"
description: "How application administrators review and approve or close disclosure case requests."
---

Application administrators approve or close disclosure case requests submitted by auditors. Approval activates scoped access for the requested period, disclosure fields, and assigned auditors.

## Prerequisites

- Access to an application workspace.
- `cases:approve_creation`.

## Workflow

<Steps>
  <Step title="Open Disclosure Requests">
    Sign in and select the application workspace. In the sidebar, click **Disclosure Requests**.
  </Step>
  <Step title="Review pending requests">
    Open the pending list. Each row represents a `disclosure_requests` record and its case-specific scope.
  </Step>
  <Step title="Open request details">
    Review request reason, period, access window, requested fields, contract filters, and requested auditors.
  </Step>
  <Step title="Approve or close">
    Approve the request to create an active case, or close it without granting access.
  </Step>
</Steps>

## Approval effects

When a request is approved:

- A `request_decisions` row is written.
- A `cases` row is created.
- Request-time auditor assignments are copied to `case_auditor_assignments`.
- Assigned auditors can open the case while the access window is active.
- Activity is recorded in `auditors_log` after the successful handler response.

## Request statuses

| Status | Meaning |
| --- | --- |
| `pending` | Awaiting administrator action |
| `approved` | Case created; assigned auditors can access scoped data |
| `closed` | Request closed without active access |

Multi-approval configurations may require additional approvers before the case becomes active.
