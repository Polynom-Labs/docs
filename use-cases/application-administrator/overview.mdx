---
title: "Application Administrator Use Cases"
description: "Application administrator workflows for disclosure decisions, case auditors, reports, and application activity."
---

Application administrators work inside an application workspace. They approve or close disclosure requests and manage case-level auditor access.

## Main use cases

| Use case | Required permission | Page |
| --- | --- | --- |
| Review disclosure requests | `cases:approve_creation` | [Approve a disclosure request](/use-cases/application-administrator/approve-disclosure-request) |
| Approve or close a disclosure request | `cases:approve_creation` | [Approve a disclosure request](/use-cases/application-administrator/approve-disclosure-request) |
| Manage case auditors | `cases:edit` | [Manage case auditors](/use-cases/application-administrator/manage-case-auditors) |
| View application activity log | `logs:view_activity` | [Application reports and activity logs](/use-cases/application-administrator/application-reports-and-activity-logs) |
| List/download application case reports | `reports:list`, `reports:download` | [Application reports and activity logs](/use-cases/application-administrator/application-reports-and-activity-logs) |

## Primary route family

```text
/workspace/application/:foreignId/disclosure
/workspace/application/:foreignId/cases/:caseId
/workspace/application/:foreignId/reports
/workspace/application/:foreignId/log
```

Application administrators operate inside one application at a time. Organization-level application creation and team management belong to [Organization Administrator Use Cases](/use-cases/organization-administrator/overview).
