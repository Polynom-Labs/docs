---
title: "Auditing Architecture"
description: "Backend services and enforcement surfaces for the Arcane Auditing Portal."
---

The Auditing Portal is separate from the user transaction path. It supports scoped disclosure, case review, reports, permissions, and activity logs.

## Runtime areas

| Area | Purpose |
| --- | --- |
| Authentication | Identifies portal users and organization context |
| Applications | Scopes disclosure and reports to partner applications |
| Cases | Stores disclosure requests, approvals, assignments, and review state |
| Reports | Generates, lists, and downloads approved reports |
| Activity log | Records review and reporting activity |
| Scanner and interpretation | Reads registered transaction evidence and prepares normalized review records |

## Enforcement surfaces

Server-side enforcement uses:

| Mechanism | Purpose |
| --- | --- |
| Auth guard | Verifies authenticated session or JWT |
| Permissions guard | Checks required permission keys |
| Application scope guard | Resolves application scope and prevents cross-application reads |
| Case assignment checks | Restrict case review to assigned auditors |
| Case access-window checks | Enforce disclosure expiration |
| Query-level filters | Prevent cross-organization and cross-application reads |

Client-side permission checks shape navigation. API guards and scoped queries are the source of truth.
